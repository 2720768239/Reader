# Mobile Reader Progress Log

> 记录当前 Reader 项目的实现进度、已验证结果、已知偏差和下一步工作。

## 当前阶段

日期：2026-06-22  
目标：把本地 Claude Blog 翻译内容做成移动优先的电子墨水阅读器，主打英文阅读、点击段落查看中文翻译、弱化页面装饰感。

## 已完成

- 文章列表页和文章阅读页已完成基础重构，整体方向从普通网页转向纸感阅读器。
- 文章列表和正文都已经去掉卡片式视觉，改为更平面的版式。
- 顶部设置入口已经收进导航栏，`Text size` 和 `Spacing` 不再出现在标题下方。
- 点击设置抽屉外的空白区域可以关闭抽屉，不再依赖 `Close` 按钮。
- 顶部导航栏支持下滑隐藏，点击正文空白区域后可以重新显示。
- 底部阅读进度已经按最新要求移除。
- 文章标题、日期/分类、正文段落、引文和纸感背景都已做过一轮视觉收敛。
- 字体大小和行距设置已恢复生效，按钮切换会直接影响正文排版。

## 已验证

- `tsc --noEmit` 通过。
- `vitest` 通过。
- `vite build` 通过。
- 阅读页交互回归测试覆盖了：
  - 段落点击后打开翻译
  - 设置抽屉打开与关闭
  - 顶栏滚动隐藏与空白处显示
  - 字号和行距选项切换
- 组件测试覆盖了：
  - 标题与日期顺序
  - 图片路径从站点根路径解析

## 当前已知问题

- 文章图片资源还没有真正导入到仓库里，所以当前页面仍然看不到图片内容。
- 目前的纸张质感和标题比例已经接近目标，但和参考图相比仍有细节差异，尤其是标题块比例、顶部抽屉停靠感和整体暖灰色调。
- 原始设计文档里提到的底部进度栏已经不再保留，当前实现以阅读优先为准。

## 下一步建议

1. 导入源图片资源，并确保 `images/...` 路径能在本地和构建产物中稳定访问。
2. 继续微调文章标题区的字号、行宽、上下留白和日期层级。
3. 再压一轮顶部导航和设置抽屉的边距、阴影和层次，尽量贴近参考图的平面感。
4. 图片导入完成后，再补一轮视觉检查，确认插图不会被背景和混合模式吞掉。
